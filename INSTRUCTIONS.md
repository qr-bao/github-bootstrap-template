# Instructions

## Startup Flow

1. Read `README.md`.
2. Read `RULES.md`.
3. Read `MACHINE.md`.
4. Apply the machine-specific settings.
5. Make only safe changes.
6. If there is a conflict, stop and ask the user.
7. Write the result to `STATUS.md`.

## AI Behavior

- Treat the repo as GitHub-first.
- Read before write.
- Prefer simple, repeatable steps.
- Keep the setup portable across computers.

## Goal

Any new computer should be able to read these files and know how to start.
